1. Business Requirements (Бізнес Вимоги)
1.1. Background (Передумови)
Проект спрямований на розробку системи для зручного доступу до послуг 3D-друку
через мережу автоматів, призначених для встановлення в місцях з високим потоком
людей, таких як університети, торгові центри та техно-хаби. Користувачі зможуть
вибирати, налаштовувати та друкувати моделі з мінімальними зусиллями, що
покращить досвід самообслуговування та зробить 3D-друк доступнішим для широкого
кола споживачів.
1.2. Business Opportunity (Бізнес-можливості)
На ринку 3D-друку бракує доступних опцій для самообслуговування у громадських
місцях. Цей проект має на меті заповнити цю прогалину, забезпечуючи зручні рішення
для друку, які сприятимуть залученню нових користувачів до технології 3D-друку.
Система орієнтована на зони з високим потоком людей, заохочуючи використання
особами, які, можливо, не мають легкого доступу до 3D-друку.
1.3. Business Objectives and Success Criteria (Бізнес-цілі та критерії
успіху)
Цілі:
● Створити масштабовану мережу автоматів для 3D-друку в зонах з високою
відвідуваністю.
● Знизити витрати на обслуговування за допомогою автоматизованих діагностик
та моніторингу статусу обладнання в реальному часі.
Критерії Успіху:
● Успішне розгортання автоматів у 3 пілотних локаціях протягом шести місяців.
● Середня кількість використань — 500 друків на місяць для кожного автомату.
● Досягнення 95% часу безвідмовної роботи для кожного автомату.
1.4. Customer or Market Needs (Потреби клієнтів та ринку)
Клієнти потребують зручної системи для завантаження та друку 3D-моделей. Система
повинна пропонувати зрозумілі опції для налаштування моделей, інтуїтивний
інтерфейс для відстеження замовлень і безпечні варіанти оплати. Також передбачено
функціонал для збереження налаштувань та історії для постійних користувачів.
1.5. Business Risks (Бізнес-ризики)
● Конкуренція на ринку: Зростання конкуренції у сфері автоматів для
самообслуговування може обмежити проникнення на ринок.
● Операційні витрати: Високі витрати на обслуговування або технічні проблеми з
автоматами можуть негативно вплинути на прибутковість.
● Прийняття користувачами: Відсутність знайомства з технологією 3D-друку
може відлякати деяких користувачів.
2. Vision of the Solution (Концепція рішення)
2.1. Vision Statement (Окреслення концепції)
Програмна система для мережі автоматів 3D-друку зробить 3D-друк доступнішим і
зручнішим у використанні, дозволяючи користувачам завантажувати моделі, вибирати
матеріали та друкувати об’єкти за запитом. Система сприятиме залученню до
технології 3D-друку та надасть інноваційний досвід самообслуговування у громадських
місцях.
2.2. Major Features (Головна функціональність)
● Управління Користувачами: Створення облікових записів, управління
профілем та підтримка ролей (користувач, адміністратор).
● Обробка Замовлень: Завантаження файлів STL/OBJ, пріоритетна обробка
замовлень, розрахунок вартості залежно від розміру моделі та матеріалу, а
також відстеження статусу замовлення.
● Управління Мережею Автоматів: Моніторинг статусу автоматів у реальному
часі, віддалене оновлення налаштувань і діагностика.
● Інтеграція Оплати: Підтримка різних платіжних шлюзів і функціонал гаманця
для постійних користувачів.
● Аналітика та Звітність: Статистика використання, журнали помилок та
фінансові звіти для оптимізації та обслуговування.
2.3. Assumptions and Dependencies (Припущення та залежності)
● Передбачається постійна доступність якісних матеріалів для 3D-друку.
● Залежить від надійної IoT-технології для моніторингу автоматів у реальному
часі.
● Передбачається стабільна інтеграція з платіжним шлюзом для обробки
транзакцій.
3. Scope and Limitations (Рамки та обмеження)
3.1. Scope of Initial Release (Рамки первинного випуску)
Основні Функції:
● Базове управління обліковими записами з підтримкою ролей.
● Завантаження файлів, налаштування параметрів і обробка замовлень.
● Моніторинг статусу автоматів і віддалене управління.
● Обробка платежів із базовою функціональністю гаманця.
Розгортання: Перший етап розгортання передбачає встановлення 3 автоматів у
ключових громадських місцях.
3.2. Scope of Subsequent Release (Рамки наступних випусків)
● Розширений інтерфейс із попереднім переглядом моделі та покращеними
можливостями налаштування замовлення.
● Додаткова аналітика, включаючи аналіз тенденцій і прогнозування споживання
матеріалів.
● Додаткові платіжні опції, включаючи криптовалюту та систему лояльності.
3.3. Limitations and Exclusions (Обмеження та винятки)
● Інструменти для редагування моделей у реальному часі не включені у
початковий випуск.
● Інтеграція з зовнішніми бібліотеками 3D-моделей планується на наступні
випуски.

4.3. Operating Environment (Робоче середовище)
Система розрахована на роботу в різних локаціях з високим трафіком, вимагаючи
стабільного підключення для інтеграції з IoT та забезпечення безпеки транзакцій.
Основні технології:
● Серверна частина: Node.js із фреймворком Nest.js для обробки API-запитів,
управління бізнес-логікою та підтримки масштабованості.
● База даних: PostgreSQL для зберігання структурованих даних, що забезпечує
ефективне оброблення запитів та взаємозв’язок даних.
● Веб-інтерфейс: React.js для динамічного та чутливого інтерфейсу користувача,
що дозволяє керувати обліковими записами, оформлювати замовлення та
відстежувати прогрес.
● Мобільна платформа: Kotlin для Android та Swift для iOS для надання зручного
інтерфейсу з можливістю отримувати сповіщення та керувати замовленнями.
● Інтеграція з IoT-пристроями: Моніторинг у реальному часі та статус автоматів
за допомогою протоколу MQTT для надійної взаємодії з автоматами.
● Обробка Платежів: Інтеграція з PayPal API (або аналогічною платформою) для
безпечних транзакцій та функціоналу гаманця для частих користувачів.
● Хмарні Сервіси: Розгортання на AWS або подібній платформі для високої
доступності, масштабованості та надійності, з використанням AWS Lambda для
безсерверних задач.
● Аналітика Даних: ELK Stack (Elasticsearch, Logstash, Kibana) для моніторингу та
аналітики в реальному часі, що забезпечує розуміння використання автоматів та
поведінки користувачів.
Ця конфігурація забезпечує надійність, безпеку та можливість обробляти великий обсяг
одночасних запитів у розподіленому середовищі
